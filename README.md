# Webpack Sample

This repository contains a minimal React application bundled with **webpack**. The app demonstrates how to lazy load a component using `React.lazy` and `React.Suspense`.

## Prerequisites

This project requires Node.js and npm. If they are not installed, download the LTS release from [nodejs.org](https://nodejs.org/). You can also install via [nvm](https://github.com/nvm-sh/nvm):

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.4/install.sh | bash
nvm install --lts
nvm use --lts
```

Confirm installation with:

```bash
node --version
npm --version
```

## Setup

Install dependencies:

```bash
npm install
```

## Development server

Start a dev server with hot reloading:

```bash
npm start
```

The application will open in your default browser on [http://localhost:9000](http://localhost:9000).

## Build

After installing dependencies you can create a production build. The compiled
files are written to the `dist` directory.

```bash
npm run build
```

Once the command completes, `dist` will contain the main bundle (`bundle.js`)
and any additional chunks generated by webpack. For this sample you should see
`src_LazyComponent_jsx.bundle.js` which contains the lazily loaded component.

These files can be served by any static file server or deployed to a hosting
provider of your choice.

## How it works

Webpack processes the `src/index.jsx` entry point and its dependencies. When the
bundler encounters `React.lazy`, it translates the lazy import into a dynamic
`import()` call. During the build this causes webpack to emit an extra JavaScript
chunk for the lazily loaded module. Without `React.lazy` everything would be
included in `bundle.js`, but with it you will see an additional file such as
`src_LazyComponent_jsx.bundle.js`. The browser only downloads this file when the
component is rendered, reducing the amount of JavaScript needed on initial page
load.
